# ParaBank Quality Assurance Software Testing Repository [1]

A comprehensive, end-to-end Quality Assurance and Software Testing suite designed for the ParaBank web application [1]. This repository encompasses automated testing, performance testing, and manual test case documentation, providing a robust framework for validating application functionality, stability, and performance [1].

## Key Features
Based on the repository architecture, this project includes:
* **Automated Test Suite**: Scalable automated tests utilizing a Java-based framework managed by Maven [1].
* **Performance Testing**: Dedicated scripts to simulate user load and analyze the application's responsiveness under stress [1].
* **Manual Testing Documentation**: Thorough manual test cases and scenarios designed to ensure comprehensive coverage of the application's features [1].
* **Detailed Reporting**: Automated generation of test execution reports using HTML, JavaScript, and CSS for clear, visual insights into test results [1, 2].

## Tech Stack & Technologies Used
* **Primary Language**: Java (14.6% of the codebase) [2].
* **Build Automation & Dependency Management**: Maven (configured via `pom.xml`) [1].
* **Reporting & UI Assets**: HTML (56.7%), JavaScript (22.9%), CSS (3.5%), and Less (2.3%) utilized for detailed test-output reporting dashboards [2].
* **Development Environment**: IntelliJ IDEA (indicated by the `.idea` configuration folder) [1].
